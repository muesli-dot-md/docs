# Muesli Docs 🥣

Documentation site for **Muesli** — *Google Docs for Markdown files*. Real-time
multiplayer markdown editing, self-hostable and open source.

This repo builds and serves **[docs.muesli.md](https://docs.muesli.md)**. The main
marketing site (**[muesli.md](https://muesli.md)**) lives in a separate `website/`
repo.

Built with [Docusaurus](https://docusaurus.io/) (classic preset, TypeScript).

> **Note:** The documentation content here is intentionally minimal for now — just
> a placeholder landing page. The real docs are written by a separate docs-content
> pass. The visual theme is also minimal and will be restyled separately.

## Develop

Requires Node.js >= 20.

```bash
npm install       # or: npm ci
npm start         # dev server with hot reload at http://localhost:3000
```

## Build

```bash
npm run build     # static site output in ./build
npm run serve     # preview the production build locally
```

## Deploy

The site ships as a small nginx Docker image published to the GitHub Container
Registry (GHCR) on every push to `main` (and on `v*` tags) via
[`.github/workflows/docker.yml`](.github/workflows/docker.yml).

Image: `ghcr.io/muesli-dot-md/docs:latest`

### Run behind Traefik on a VPS

Prerequisites on the host:

- An external Docker network named `traefik` (`docker network create traefik`),
  shared with your Traefik container.
- A running Traefik with a `websecure` (:443) entrypoint and an ACME cert
  resolver named `letsencrypt` (adjust label values in `docker-compose.yml` if
  yours differ).

Then:

```bash
docker compose pull
docker compose up -d
```

Traefik terminates TLS and routes ``Host(`docs.muesli.md`)`` to the container's
port 80. No host ports are published by this service.

### Build the image locally

```bash
docker build -t ghcr.io/muesli-dot-md/docs:latest .
docker run --rm -p 8080:80 ghcr.io/muesli-dot-md/docs:latest
# open http://localhost:8080
```

## Layout

- `docs/` — Markdown documentation pages (served at the site root, `/`).
- `docusaurus.config.ts` — site configuration (URL, navbar, footer).
- `sidebars.ts` — autogenerated sidebar from `docs/`.
- `src/css/custom.css` — theme overrides (kept minimal).
- `Dockerfile`, `nginx.conf`, `.dockerignore` — container build + serving.
- `docker-compose.yml` — Traefik deployment example.
- `.github/workflows/docker.yml` — CI: build + push to GHCR.
